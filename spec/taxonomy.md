# OpenPAKT — Finding Taxonomy

## Overview

This document defines the **OpenPAKT finding taxonomy**, which provides a standardised classification system for AI agent security risks.

The taxonomy enables consistent reporting and interpretation of security findings across tools and platforms.

## Design Goals

The taxonomy is designed to:

- normalize common AI agent security risks
- support consistent reporting across scanners
- align with established security guidance
- enable deterministic CI policy evaluation

## Specification

The taxonomy defines standardised identifiers for security findings such as prompt injection, tool misuse, and data exfiltration.

Detailed taxonomy definitions will be introduced in future revisions.

## Examples

Example taxonomy usage can be found in the example reports provided in the `examples/` directory.

## Compatibility Considerations

The taxonomy is designed to align with widely recognized AI agent security guidance and risk classifications.
